软件安装

源码安装

  • 从可信渠道获得软件源码包，通常是tar.gz或tar.bz2的压缩文件

    wget {URL}/package.tar.gz

  • 解压：tar命令解压源码包

  • 配置：运行./configure脚本进行安装配置和依赖检查，生成Makefile文件

  • 编译：make指令

  • 安装：make install生成可执行文件、库、配置文件等复制到系统指定位置

  • 清理：make clean清理编译过程的临时文件

  • 卸载：make uninstall

Debian系列软件包安装

  • 通常使用.deb格式，使用dpkg工具安装

  • dpkg仅能安装单个软件包，无法处理多个软件包之间的依赖和版本问题，若被依赖的其他软件包未安装则会出错

Redhat系列软件包

  • .rpm软件包，通过rmp工具安装

软件源安装

  • apt是建立在dpkg上的安装工具

    • 设置软件源：配置文件路径：/etc/apt/source.list。最好改为国内软件源

    • 软件包更新：apt update

    • 查询软件包：apt search

    • 安装软件包：apt install

    • 卸载软件包：apt remove

  • yum是基于rmp上的安装攻击

    13. 输入输出重定向

      • 将命令标准输入或输出重定向到别的文件

      • 文件是IO的基本抽象，标准输入、输出和出错可以替换为其他文件

      • 命令 > 文件 重定向标准输出到文件，覆盖

      • 命令 >> 文件 重定向标准输出到文件，追加

      • 命令 < 文件 重定向标准输入到文件，从文件读取

      • 命令 << 字符（常用EOF） 从标准输入读取数据，当碰到EOF字符串时结束，常用其他重定向共用

      • 命令<<<输入 读取输入到命令

      • 命令 2> 文件、命令 2>> 文件 对应标准出错的重定向

      • 命令 &> 文件、命令 &>>文件 同时重定向标准输出和标准出错（文件只被打开一次）

正则表达式

  1. 用途：RE正则表达式与正则文法等价，完全对应有限状态机

    • 进行模式匹配

  3. 正则表达式的语法

    1. 单个匹配

      1. .匹配任意单个字符

      2. []匹配任意单个指定范围内的任意字符，可用“-”连接连续字符（用边界指定范围）

      3. [^]匹配任意单个除了指定范围内的任意字符

      4. \s匹配任意单个空白符

      5. \w匹配任意单个字母或数字或下划线

    2. 数量匹配

      1. *前面的字符出现0次及以上

      2. ?前面的字符出现0次或一次

      3. +前面的字符出现1次及以上

      4. {n}前面的字符出现n次

      5. {n,m}前面的字符出现的次数在n,m之间

      6. {n,}前面的字符至少出现n次

      7. {.n}前面的字符至多出现n次

      8. ps:基本正则表达式使用{}需要用\转义，拓展正则表达式匹配“{}”需要使用\转义

    3. 位置锚定

      1. ^[]匹配行首

      2. []$匹配行尾

      3. \b匹配单词

    4. 其他

      1. | 表示或，可以用（）改变优先级，本身优先级最低

      2. () 分组，将一个或多个字符捆绑到一起，当作一个整体处理,分组中的模式所匹配到的内容会被RE自动编号

      3. \ [num]使用分组

    5. 贪婪匹配和懒惰匹配

      1. 默认贪婪匹配，匹配尽可能多的字符

      2. 命令之后加 ? 懒惰匹配

文件系统原理

物理组织结构

  1. 链式结构：文件信息存放在若干不连续的物理块中，块间链接

  2. 顺序结构

  3. 索引结构：文件信息存放在若干不连续的物理块中，文件关联一个索引块，记录索引关系（经典UNIX结构：索引块inode）

UNIX文件系统原理

  1. 典型UNIX磁盘布局（ext2文件系统）

  2. 普通文件的实现

    1. UNIX普通文件采用索引结构-inode

    2. inode存储所有元数据

    3. 特殊的索引方式：共13个索引：直接索引x10（直接指向块），一级间接索引x1（指向的索引块指向块），二级间接索引x1（指向的索引块指向的索引块指向块），三级间接索引x1（……）

  3. 目录文件：

    1. 目录也是文件，为有特殊结构的文件

    2. 自动维护目录结构，添加保护防止误操作（用户不能修改其中的内容）

    3. 链接数（硬链接）：

      • 允许一个inode节点对应多个文件名，所有文件地位平等，删除任意一个不会对其他文件产生影响

    4. 软链接：

      • 本身就是一个文件，有自己的indoe和数据块

      • 数据块中储存了另一个文件或目录的路径，而不是文件或目录的实际数据

  4. 文件系统的挂载：

    1. 挂载点：文件系统挂载的位置

    2. 一个文件系统（新的存储空间）必须挂载（mount）之后才能使用

    3. UNIX中允许其文件系统中的任意目录成为挂载点，但会屏蔽原有内容，直到挂载文件系统被卸载

  5. 磁盘操作：

    1. fdisk -l 列出系统所有磁盘的分区表

    2. fdisk 用来分区指令

    3. mkfs 用于格式化分区

    4. umount卸载已挂载的文件系统，umount [选项] 『目录|设备』

    5. lsblk 列出块设备的信息

    6. df 显示每个指定文件所在的文件系统磁盘的信息

    7. du [选项] [文件] 显示目录或文件的磁盘使用情况，显示磁盘的占用情况

      • [-d]设置目录文件的递归深度，默认递归所有子文件

      • [-h]以易读方式显示

    8. fsck检查并修复文件系统

    9. mountpoint检查目录是否为挂载点

    10. dd 对文件进行基于块的复制和转换

      • dd [操作]

      • 所有[操作]均以赋值方式给出

        • if 读入文件或设备

        • of 写出的文件或设备

        • bs 读和写的块大小

        • count 读入的块数量

        • skip 读入巍峨恩间时跳过的块数量

        • seek 写出文件时从对应的块数量位置开始

  6. 其他常用命令

    1. cut 纵向输出文件的某个列

    2. paste 横向链接多个文件并输出

    3. sort 对多个文件按行排序并输出，原文件不变（默认字典序排序）

    4. find文件搜索命令：

      • find [-L] [目录] [表达式]

      • 从指定目录遍历目录树，搜索使得表达式为真的所有文件

        • -type 对应的文件类型

进程

进程是运行中的程序

进程结构

  1. 进程结构（Linux IA32）

  2. 虚拟内存

  3. 进程状态：进程会在生存周期中经历多次进程状态的转换（进程状态机）

进程查看常用方法

  1. ps [选项] 进程查看命令，默认当前用户的进程

    • -f显示进程完整信息

    • -e显示所有进程

    • -o指定输出列

    • -H按树结构显示

    • -p指定进程号

    • -u指定用户

  2. top [-d num]进程查看指令

    • -d指定刷新间隔/s

    • htop显示方式较top更友好

  3. /proc文件系统

    • /proc是Linux上一个自动挂载的伪文件系统，为用户提供了一个查看和操作内核数据结构的接口

    • cd $$进入/proc中的当前进程,$$为Linux提供的统配符，指当前进程

进程控制

  1. 进程组：一组有相同进程组ID的进程，有时一个进程组也称为一个作业

  2. 作业/进程的启动（前台进程/作业）：在shell中运行指令

  3. 后台进程/作业：$ command &

    • 不占用用户终端，不能与用户交互

    • 通常适用于批处理程序

    • 通常将后台作业的输出重定向，防止输出干扰用户

  4. 作业控制：

    • <ctrl+z>，将前台作业暂停，并切换到后台

    • bg %作业号：将被暂停的后台作业重新执行

    • fg %作业号：将后台作业切换到前台继续执行

    • 查看当前活跃的后台作业：jobs

  5. 终止进程：

    • kill PID

    • kill -9 PID强制杀死进程

  6. fork通过复制当前进程的方式，创建子进程

  7. exec系统调用族：子进程可以拥有自己的可执行代码，

  8. exit系统调用族：终止调用进程，回收进程资源

UNIX信号

  1. 信号是UNIX系统提供的一种针对进程的异步通知机制

  2. 可以由一个进程发送到另一个进程

  3. ctrl+c，实质上是shell向前台作业发送了SIGINT信号（默认终止）

  4. ctrl+z，发送SIGTSTP信号（默认暂停）

    • PS：对信号的默认响应可修改，但kill -9等发出的强制指令信号响应方式不可修改

    • trap SIGINT恢复SIGINT响应为默认处理

shell变量

  1. 根据变量的作用范围，可将变量分为：

    • 普通变量：当前shell全局有效

    • 环境变量：当前shell全局有效，且能传递给子进程的变量

    • 局部变量：在函数内定义，仅在函数内有效

  2. 设置普通变量

    1. shell变量的唯一类型：字符串

      1. 变量名=变量值（等号两侧不能有空格）

      2. 若变量值字符串中包含空白字符，需要用""/''括起来

      3. 若变量未定义，则定义变量，若已定义，则重新赋值

    2. 查看和删除变量

      1. 查看一定义的变量：set

      2. 删除变量：unset 变量名

      3. 读入变量：read 变量名 ...，从标准输入中读入一行字符串，对该字符串基于空白符分割，然后将分割后的字串存入变量中（读入的一行字符串中的空白符都会被去重）

  3. 设置环境变量：

    1. export 变量名[=变量值]

  4. 引用变量时的变量变换：

    1. 使用${变量名}

    2. 支持特殊表达式

  5. 标准变量：

    1. PATH：指定shell外部命令的搜索路径和搜索顺序，该标准变量会影响shell对外部命令的执行

    2. PS1/PS2：

      • 指定作为命令提示符的字符串

      • PS2出现在用户尚未键入完整命令但是键入之后，用于告知用户继续输入

    3. IFS：在进一步命令行解析时，用于指定对替换后展开的文本进行进一步单词分割时的分隔符

    4. $：$$保存当前进程PID

shell元字符

在shell中，元字符是一些具有特殊含义的字符

  1. \<元字符> 转义

  2. ~[用户名] 主目录

  3. ；shell间隔命令

  4. (命令;命令;)将一组命令作为单个命令，在一个新的子shell中执行

  5. '命令'执行命令或命令组，将命令行中的该元字符原有位置'命令'替换为执行结果

  6. $()与#6相同，可读性更强

  7. 算数表达式替换：$((算数表达式))

  8. 字符串引用：

    • 单引号：所有字符均被视为普通字符，不进行转义

    • 双引号：会进行转义、变量替换、命令替换和算数表达式替换

  9. 路径通配符[*]/[?]/[1-9]...：输入路径时，用于匹配文件

  10. 逻辑运算符：

    • 与一般情况不同的是：0表示true;!0表示false

    • &&执行短路操作
